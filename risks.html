Risk

AR Risk

Working with AR can be extremely risky because it is a new technology and can be easy to overestimate its capabilities and underestimate just how difficult it might be to create working prototypes. AR is taking off but because it is new it might lack in available learning resources especially for use on phones.

Unity Risk

Unity is an extremely useful tool, but it can be complex, hard to learn and very overwhelming. When using unity, it might be hard to publish a finished project because many people spend years learning it and still find aspects of it challenging.

C# Risk

Learning any coding language is hard and can be extremely time consuming but C# is the choice of language when working with unity and it is known for being hard to understand and complex. When working with code small errors like misspelling and misplacing symbols can stop even a basic function from working and not having enough experience to see the error can have one simple mistake ruin hours of work.

Hardware Risk

Programming a game on android can be easily done with almost any computer but when using IOS a mac is needed to write XCode, and this can be costly and was unforeseen until the team started designing the project. Having hardware powerful enough to run the programmes used to develop games can also be expensive.
